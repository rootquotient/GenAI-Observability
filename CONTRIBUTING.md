# Contributing Guide

Thank you for your interest in contributing to GenAI API Cost & Drift Monitor.
We welcome contributions of all kinds, including bug reports, documentation,
and feature improvements.

---

## Project philosophy

- Provider-agnostic by design
- Privacy-first defaults
- Observability, not control (for MVP)
- Simple abstractions over complex behavior

Please keep these principles in mind when contributing.

---

## How to contribute

### 1. Reporting issues

If you find a bug or have a feature request:

- Check existing issues before opening a new one
- Clearly describe the problem or proposal
- Include relevant context or examples if possible

---

### 2. Making changes

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes with clear commit messages
4. Add or update documentation if needed
5. Ensure the project builds successfully
6. Open a pull request

---

### 3. Pull request guidelines

- Keep PRs focused and small
- Avoid unrelated refactors
- Explain _why_ the change is needed
- Link related issues if applicable

---

## Code style

- Use TypeScript strict mode
- Prefer explicit types over `any`
- Avoid premature abstractions
- Keep functions small and single-purpose

---

## Development setup

Basic setup steps are documented in the README.
No external services are required beyond PostgreSQL for local development.
